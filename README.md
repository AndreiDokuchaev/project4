# Интеграция нового источника данных и построение инкрементальной витрины по заказчикам в DWH для маркетплейса товаров ручной работы
Цель проекта: интегрировать данные нового источника в существующее DWH и создать инкрементальную витрину для поддержки программы лояльности, персонализации рекомендаций и корректировки маркетинговых акций — с фокусом на поведении и расходах заказчиков.

Задачи проекта:

Подключиться к целевой БД.

Проанализировать данные в таблицах нового источника и корректно распределить их по соответствующим измерениям и фактам в схеме dwh.

Дополнить существующий скрипт загрузки данных новым источником, обеспечив регулярный инкрементальный забор данных с учётом того, что сайт‑источник будет функционировать в переходный период.

На основании требований бизнес‑аналитика спроектировать DDL‑структуру витрины: определить типы данных, названия полей, добавить комментарии и предусмотреть таблицу с датой загрузки для инкрементальности.

Разработать скрипт инкрементального расчёта витрины, реализующий требуемые метрики: суммарные траты и доход платформы, количество и статусы заказов, среднюю и медианную длительность исполнения, популярные категории и мастеров, а также месячную разбивку отчётного периода. За основу взять логику витрины по мастерам.

Используемые инструменты:

SQL

PostgreSQL

Вывод:
В результате выполнения проекта новый источник данных успешно интегрирован в DWH: данные из схемы external_source распределены по соответствующим таблицам измерений и фактов. Создана инкрементальная витрина, содержащая комплексные метрики по заказчикам — включая траты, статусы заказов, популярные категории и мастеров, а также расчёт дохода платформы. Скрипт инкрементального обновления витрины обеспечивает актуальность данных без полной перезаписи. Результаты проекта можно считать удовлетворительными: все поставленные задачи выполнены, витрина протестирована и готова к использованию для задач программы лояльности, рекомендаций и маркетингового планирования.
